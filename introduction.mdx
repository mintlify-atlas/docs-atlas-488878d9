KeyForge is a browser-based password generator that uses your device's cryptographically secure random number generator to produce passwords that are statistically impossible to predict.

## The problem with most password generators

Many online password generators rely on `Math.random()` — JavaScript's built-in pseudo-random number generator. `Math.random()` is designed for speed and statistical distribution, not security. Its output is seeded from a predictable source and can, in theory, be reverse-engineered by an attacker who observes enough output.

<Note>
  KeyForge uses `crypto.getRandomValues()` — the Web Cryptography API's CSPRNG (cryptographically secure pseudo-random number generator). This is the same primitive used by TLS handshakes and cryptographic key generation. Every character selection and every shuffle is driven by `crypto.getRandomValues()`, with no fallback to `Math.random()`.
</Note>

After sampling the random bytes, KeyForge shuffles the character pool using a Fisher-Yates algorithm also driven by `crypto.getRandomValues()`. This guarantees that the final order of characters is as unpredictable as the characters themselves.

## How KeyForge differs from typical generators

Most password generators make two decisions: which characters to include, and how many. KeyForge adds a third layer — it actively tests each candidate password against a set of anti-pattern rules before accepting it:

- **Keyboard walk detection** scans the output against all standard QWERTY keyboard rows, forward and backward.
- **Sequential character detection** rejects runs of consecutive ASCII code points in either direction.
- **Weak fragment matching** checks against a curated list of dictionary fragments and common sequences (e.g. `hello`, `secre`, `qwer`).
- **Consecutive repeat detection** optionally rejects any character that appears twice in a row.

If a generated candidate fails any enabled rule, the engine discards it and generates a new one, up to 200 attempts. This means you always receive a password that passes every guard you have enabled.

The Passphrase mode draws from a 200-word curated EFF-style wordlist. The Phonetic Shield mode assembles passwords from onset–vowel–coda syllable structures, interleaving entropy characters between syllable groups and appending additional entropy at the end to satisfy common policy requirements.

## Architecture

KeyForge is a single HTML file with three accompanying JavaScript modules:

| File | Responsibility |
|---|---|
| `pasgen.js` | Core generation engine: password, passphrase, PIN, and Phonetic Shield algorithms |
| `entropy.js` | Entropy calculation, crack-time estimation, strength levels, and DNA rendering |
| `ui.js` | DOM event wiring, settings reads, live regeneration on setting change, theme management |

<Info>
  KeyForge has no backend, no API calls, and no analytics. Every computation runs entirely in your browser. The only data written outside of memory is your theme preference, stored under the key `kf-theme` in `localStorage`. Your generated passwords never leave your device.
</Info>

Because there is no server involved, KeyForge works offline after the initial page load. You can save the page locally or use it directly at [https://keyforgegd.vercel.app](https://keyforgegd.vercel.app).

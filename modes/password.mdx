The password generator builds random passwords using the browser's `crypto.getRandomValues()` API — a cryptographically secure pseudo-random number generator (CSPRNG). No server calls are ever made; every password is generated entirely in your browser.

## How the generator works

<Steps>
  <Step title="Build the character pool">
    Based on your selected character classes (lowercase, uppercase, digits, symbols), KeyForge assembles an allowed character pool. Excluded characters (ambiguous or custom) are filtered out before any sampling begins.
  </Step>
  <Step title="Guarantee minimums">
    Before random fill, the generator inserts the required number of characters from each enabled class (e.g. at least 1 uppercase, 1 digit). This ensures policies like "must contain a digit" are always satisfied.
  </Step>
  <Step title="Fill remaining positions">
    The remaining slots are filled by sampling uniformly at random from the full pool using the CSPRNG.
  </Step>
  <Step title="Fisher-Yates shuffle">
    The entire character array — guaranteed minimums and random fill combined — is shuffled with a CSPRNG-backed Fisher-Yates algorithm. This eliminates any positional bias from the guarantee step.
  </Step>
  <Step title="Anti-pattern guard loop">
    If any enabled guard detects a prohibited pattern, the attempt is discarded and the process restarts. Up to 200 attempts are made. If all 200 fail, the guards are relaxed and a clean random result is returned as a fallback.
  </Step>
</Steps>

## Settings reference

<ParamField path="options.length" type="number" default="16">
  Total password length. Range: 8–128 characters.
</ParamField>

<ParamField path="options.useLower" type="boolean" default="true">
  Include lowercase letters `a–z` in the pool.
</ParamField>

<ParamField path="options.useUpper" type="boolean" default="true">
  Include uppercase letters `A–Z` in the pool.
</ParamField>

<ParamField path="options.useDigits" type="boolean" default="true">
  Include digits `0–9` in the pool.
</ParamField>

<ParamField path="options.useSymbols" type="boolean" default="true">
  Include symbols from the configured symbol pool.
</ParamField>

<ParamField path="options.symbolSet" type="string" default="!@#$%^&*()_+-=[]{}|;:,.<>?/~">
  The set of symbol characters available when symbols are enabled. You can replace this entirely via the **Symbol pool** input.
</ParamField>

<ParamField path="options.excludeAmbiguous" type="boolean" default="false">
  Remove visually ambiguous characters from every enabled class: `0 O o l 1 I | \` ' " \\`
</ParamField>

<ParamField path="options.noRepeating" type="boolean" default="false">
  Reject any password that contains two identical adjacent characters (e.g. `aa`, `11`).
</ParamField>

<ParamField path="options.noSequential" type="boolean" default="true">
  Reject passwords containing keyboard-row walks (`qwe`, `asd`, `123`) or sequential character runs (`abc`, `xyz`) of 3 or more characters, in either forward or reverse direction.
</ParamField>

<ParamField path="options.noWeakFragment" type="boolean" default="true">
  Reject passwords that contain any of the built-in weak-pattern fragments such as `pass`, `admin`, `qwer`, `1234`, `hello`, and others.
</ParamField>

<ParamField path="options.customExclude" type="string" default="">
  A string of individual characters to remove from the pool regardless of which classes are enabled. Example: `@#&` removes those three symbols.
</ParamField>

<ParamField path="options.customInclude" type="string" default="">
  A string of characters to guarantee appear at least once. Only characters already present in the pool are included; characters not in the pool are silently ignored.
</ParamField>

<ParamField path="options.minLower" type="number" default="1">
  Minimum number of lowercase characters when lowercase is enabled. Range: 0–20.
</ParamField>

<ParamField path="options.minUpper" type="number" default="1">
  Minimum number of uppercase characters when uppercase is enabled. Range: 0–20.
</ParamField>

<ParamField path="options.minDigits" type="number" default="1">
  Minimum number of digit characters when digits are enabled. Range: 0–20.
</ParamField>

<ParamField path="options.minSymbols" type="number" default="1">
  Minimum number of symbol characters when symbols are enabled. Range: 0–20.
</ParamField>

## Anti-bruteforce guards

Guards are applied after shuffling and checked on every attempt. Enabling more guards narrows the accepted keyspace slightly but dramatically raises the cost of pattern-based attacks.

| Guard | UI label | Default | What it blocks |
|---|---|---|---|
| `noRepeating` | No consecutive repeats | Off | Any two identical adjacent characters (`aa`, `$$`) |
| `noSequential` | Block keyboard walks | On | Keyboard-row sequences (`qwe`, `asd`, `zxc`, `123`) and ascending/descending character-code runs (`abc`, `9876`) of 3+ chars |
| `noSequential` | Block keyboard walks | On | Same check also covers reverse walks (`ewq`, `fds`) |
| `noWeakFragment` | Block weak fragments | On | Substrings matching a built-in list of predictable words and patterns: `pass`, `word`, `admin`, `root`, `user`, `login`, `test`, `demo`, `temp`, `qwer`, `asdf`, `zxcv`, `1234`–`6789`, `abcd`, `hello`, `secre`, `defau`, `guest`, and more |

<Note>
Keyboard-walk detection scans all four keyboard rows: `qwertyuiop`, `asdfghjkl`, `zxcvbnm`, and `1234567890`. Both forward and reverse traversals are blocked.
</Note>

## Custom character sets

### Symbol pool

The default symbol pool is:

```text
!@#$%^&*()_+-=[]{}|;:,.<>?/~
```

Replace the entire pool by editing the **Symbol pool** field. The pool is used only when **Symbols** is checked.

### Exclude characters

Enter individual characters in the **Exclude characters** field to remove them from every character class. This is applied on top of the ambiguous-character filter. Example: type `@$` to prevent both `@` and `$` from ever appearing.

### Force include

The **Force include** field guarantees that each listed character appears at least once in the generated password. Characters not already in the active pool are ignored — enabling symbols and placing `@` in the force-include field only works if `@` is also in the symbol pool.

## Minimum character counts

The minimum counts section lets you set a floor for each character class:

- **Min uppercase** — at least this many `A–Z` characters
- **Min lowercase** — at least this many `a–z` characters
- **Min digits** — at least this many `0–9` characters
- **Min symbols** — at least this many symbol characters

The sum of all minimums must not exceed the total password length, otherwise no valid password can be produced.

## Entropy calculation

Password entropy is calculated as:

```text
entropy = length × log₂(poolSize)
```

where `poolSize` is the count of unique characters in the active pool after all exclusions.

For example, a 16-character password using all four default classes (26 + 26 + 10 + 28 = 90 characters) yields:

```text
16 × log₂(90) ≈ 16 × 6.49 ≈ 103.9 bits
```

This places it in the **Strong** tier. Enabling ambiguous-character exclusion reduces the pool size by up to 11 characters.

### Strength tiers

| Tier | Bits |
|---|---|
| Very Weak | 0–35 |
| Weak | 36–55 |
| Fair | 56–72 |
| Strong | 73–94 |
| Very Strong | 95–127 |
| Fortress | 128+ |

## Tips

<Tip>
For most accounts, 16 characters with all four character classes enabled gives over 100 bits of entropy — well into the Strong tier. There is rarely a practical reason to go below 16.
</Tip>

<Tip>
If a site rejects certain symbols, use the **Exclude characters** field rather than disabling the entire Symbols class. Keeping symbols in the pool maximises entropy even with a few characters removed.
</Tip>

<Tip>
Use **Force include** when a site has a policy like "must contain `@` or `#`". Set both force-include and keep symbols enabled — this satisfies the policy while still giving the CSPRNG full control over every other position.
</Tip>

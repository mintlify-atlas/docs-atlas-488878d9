## Why KeyForge?

Most password generators rely on `Math.random()` — a pseudo-random function that is **not cryptographically secure**. KeyForge uses `crypto.getRandomValues()`, the same source of entropy used by TLS and other security-critical systems. Every character is drawn from a true CSPRNG (Cryptographically Secure Pseudo-Random Number Generator).

## Four generator modes

<Tabs>
  <Tab title="Password">
    Generate a classic random password with full control over length (8–128 characters), character classes, minimum per-class counts, custom symbol pools, and four anti-bruteforce guards.

    **Key settings:** length slider, uppercase/lowercase/digits/symbols toggles, exclude ambiguous characters, block keyboard walks, block weak fragments, custom exclude/include characters.
  </Tab>
  <Tab title="Passphrase">
    Build a passphrase from a curated 200-word EFF-inspired wordlist. Words are joined by a separator of your choice and optionally augmented with a 2-digit number, a symbol, and leet substitutions (a→@, e→3).

    **Key settings:** word count (3–10), separator (dash, dot, space, underscore, hash, none), capitalize, append number, append symbol, leet substitution.
  </Tab>
  <Tab title="PIN">
    Generate a numeric PIN with configurable length (4–20 digits) and security rules that block repeating digits, sequential runs (1234, 9876), and common PIN patterns.

    **Key settings:** PIN length slider, no-repeat digits, block sequences, block common patterns.
  </Tab>
  <Tab title="Phonetic Shield">
    Build passwords from pronounceable syllable groups (onset + vowel + optional coda) with entropy characters injected between them. A pronunciation guide helps you memorize the result.

    **Key settings:** syllable groups (2–6), entropy injection level (light / medium / heavy), capitalize syllables, show pronunciation guide.
  </Tab>
</Tabs>

## Security at a glance

| Property | Value |
|---|---|
| Randomness source | `crypto.getRandomValues()` (CSPRNG) |
| Shuffle algorithm | Fisher-Yates with CSPRNG |
| Max password length | 128 characters |
| Threat model baseline | 1 trillion guesses/second (offline GPU farm, raw hash) |
| Data collection | None |
| Server requests | None — fully offline after page load |
| Storage | `localStorage` for theme preference only |

<Note>
  KeyForge estimates crack times against an **offline** attacker using raw MD5/SHA at 10¹² guesses per second. Against properly hashed credentials (bcrypt, Argon2), the real times are orders of magnitude longer.
</Note>

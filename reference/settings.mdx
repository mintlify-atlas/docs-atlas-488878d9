    ### Anti-bruteforce guards

    <ParamField path="excludeAmbiguous" type="boolean" default="true">
      Removes visually ambiguous characters from the pool before generation: `0`, `O`, `o`, `l`, `1`, `I`, `|`, `` ` ``, `'`, `"`, `\`. Prevents copy-paste errors and read-back mistakes.
    </ParamField>

    <ParamField path="noSequential" type="boolean" default="true">
      Rejects any candidate password that contains a keyboard walk (3+ consecutive characters in the same row of a QWERTY keyboard) or sequential ASCII characters (e.g., `abc`, `789`). Also checks reverse walks.
    </ParamField>

    <ParamField path="noRepeating" type="boolean" default="false">
      Rejects any candidate password that contains two adjacent identical characters (e.g., `aa`, `11`). Off by default because it has a meaningful impact on pool diversity for shorter passwords.
    </ParamField>

    <ParamField path="noWeakFragment" type="boolean" default="true">
      Rejects passwords that contain any substring from a list of 34 weak fragments (e.g., `pass`, `admin`, `qwer`, `1234`). Checked case-insensitively.
    </ParamField>

    ### Custom character sets

    <ParamField path="customExclude" type="string" default="">

    </ParamField>

    <ParamField path="customInclude" type="string" default="">

## Global settings

### Theme

KeyForge supports dark and light themes. Click the sun/moon icon in the top-right corner of the app to toggle. Your preference is stored in `localStorage['kf-theme']` â€” the only data KeyForge ever writes to browser storage.

| Value | Description |
|---|---|
| `dark` | Dark background (default on dark-preferring OS) |
| `light` | Light background (default on light-preferring OS) |
